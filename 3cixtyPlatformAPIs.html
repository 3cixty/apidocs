3cixty Platform APIs

3cixty APIs are defined in following categories.


Query Manager/Augmenter APIs

The Query Manager/Augmenter endpoint is the main endpoint that developers can use to make queries to the 3cixty knowledgebase. Note that, these queries can also be augmented, e.g., based on the user provided ratings to an item and is based on the data available in the KB. The queries can be made using a RESTful API, or using Rich RESTful API that support SPARQL queries as a parameter to the API. The creation of the SPARQL queries can be understood using the Developer mode in the 3cixty related ExplorMI 360 app.

The result provided by the APIs is in JSON format. Various APIs are exposed includes counting items, clustering based on single filter and multiple filters, extracting items with relevant information, and providing results based on ratings provided to an item. APIs are also provided that mix multiple above mentioned API features.

The functionality of an API is only limited to reading the data from the KB. Please do not provide Insert/Delete SPARQL queries, the APIs will not work.

Please refer to the Knowledgebase schema here for the complete understanding of the 3cixty Knowledgebase.

Go to APIs

Wishlist API (also known as Tray API)

The Wishlist API is a set of APIs that provide features for reading and writing the prefered items of a user from and into the 3cixty Platform. One of the feature of the API is to provide listing of the stored items in the Wish List. The API returns this output in JSON format. Further, using other parameters required for the API, items can be added to the Wish List. The API also provides functionality for updating the items in the Wish List. The update feature is limited to whether the item was already attended by the user, time when the item was attended, rating provided by the user, and whether the item was deleted. On a successful execution, the HTTP status code = 200 is returned otherwise appropriate HTTP status code is returned.

Various actions that can be performed on the Wish List are:

  • add an item to the Wish List,
  • update an item in the Wish List,
  • retrieving all the items in the Wish List,
  • retrieve all items (information in details) in the Wish List,
  • login into Wish List,
  • remove all items from the Wish List

Further when retrieving all the items in the Wish List, a JSON is produced that has the information about the items in the Wish List. The JSON primarily include properites like: element_id, element_type, element_title, timestamp, token, source, attend, attend_datetime, rating, image_url.

Go to APIs

Profile APIs

Profile API provides features including reading and writing of the user profile in the KB. If a user logs in for the first time, the APIs store user credentials limited to 3cixty ID, First Name, Last Name, Profile Image on the source and the source used to authenticate the user in the KB. Note that here the source can be either Google or Facebook. On a successful execution the HTTP status 200 is returned otherwise appropriate HTTP status code is returned. The Profile API supports reading and writing of data fields like:

  • 3cixty UID which is unique for each user
  • First name
  • Last name

Go to APIs

OAuth APIs

The 3cixty Oauth is the Authentication and Authorization component that authenticates a user and the application associated to 3cixty Platform.

The 3cixty OAuth APIs provides capabilities to:

  • Developers to register their applications which need access to user's private data such as their Wish List

  • Users to explicitly declare whether or not they want the application to access their personal data, viz., their Wish List

  • Authenticate and Authorize user and applications

Before going into the details of APIs associated to OAuth, we describe some terms used in this document:

  • 3cixty App Key: is a unique string within the 3cixty Platform. This key is used to authenticate applications that are using the 3cixty Platform.

  • redirect_uri: is a callback URI for Web applications. Basically, when a Web application authenticates with the 3cixty Platform, the 3cixty Platform internally generates a 3cixty token and uses the redirect_uri value to send you back the 3cixty token. The 3cixty token includes information about:

    • 3cixty access_token

    • 3cixty refresh_token

    • 3cixty scope

    • expires_in

    The redirect_uri is not required by the Android applications. However, it is mandatory for the Web applications. A typical example of full URI that the 3cixty Platform sends back to the Web application is:

    http://test.com/redirect_uri.html#access_token=<access_token>&refresh_token=<refresh_token>&expires_in=<expiration_in_second>&scope=<scope>

  • 3cixty access_token: is a unique string within the 3cixty Platform. You mostly have to use the 3cixty access_token to call 3cixty Platform APIs. After being generated, each 3cixty access_token is only valid for a day (84600 seconds). Whenever your current 3cixty access_token is invalid, you have to refresh the expired token.

  • 3cixty refresh_token: is a unique string within the 3cixty Platform. The 3cixty refresh_token always accompanies a 3cixty access_token. The 3cixty refresh_token is used to refresh the expired token. The results of refreshing expired token is that it will generate a new 3cixty access_token, a new 3cixty refresh_token, but will have the same 3cixty scope and will be valid for 1 day.

  • 3cixty scope: is what users grant permission for. Please note that 3cixty scope is in a comma separated for/mat. The list below provides all the possible values for the 3cixty scope:

    • Profile

    • WishList, Profile

    • WishList

    • null

    Note: null means that users did not grant applications the access to their Profile or Wish List.

Following APIs in the OAuth APIs list can be accessed via 3cixty Platform directly to understand better:

  • GET /getGoogleAccessToken

    This API runs on a web browser to get a Google access_token. Please open https://api.3cixty.com/v2/googleToken.jsp on your web browser to get your the Google access_token.

  • Web Authentication

    • GET /auth

      This endpoint supports JavaScript applications that run on a Web browser.

      After authenticating a user with the 3cixty Platform, the 3cixty platform sends back a 3cixty token to you through the 3cixty redirect_uri. The 3cixty token includes information about 3cixty access_token, 3cixty refresh_token, expires_in and a list of accepted 3cixty scope(s).

      For example, if your redirect_uri = http://test.com/redirect_uri.html, typical values sent by the 3cixty Platform to your Web application are in the URL location like: 

      http://test.com/redirect_uri.html#access_token=<access_token>&refresh_token=<refresh_token>&expires_in=<expiration_in_second>&scope=<scope>

      To use this API developers should provide their 3cixty App Key in the query parameter of the HTTP request.

      If the 3cixty App key is invalid, you will get an error message, for example {"response": "failed", "reason": "<reason>"}.

      Some reasons returned are:

      • key is invalid

      Please check a Javascript application at https://api.3cixty.com/oauth-demo/login.html to know how to authenticate with the 3cixty Platform.

  • Android Authentication

    Please check a Android application at https://github.com/3cixty/android-client-sample to know how to authenticate with the 3cixty Platform.

MoveSmarter APIs

The MoveSmarter platform is an open ICT toolbox for App developers that creates capabilities for an App to automatically measure individual travel behaviour via the Smartphone and to create a 24x7 multimodal travel profile of a user. With this travel profile, for example, Apps can be designed that provide insight in traveller flows and patterns, modality use and transits and frequent routes of target groups. This can be done on an individual level providing users with feedback on their travel behaviour and profile or providing tailored information (on e.g. city POI's you have missed so far) based on this profile. This can also be done on a group level providing target groups of users with feedback on their collective travel behaviour or to provide information that is useful in a social context (sharing profile information).

The MoveSmarter platform can be used together with other 3cixty APIs and interfaces to create advanced application scenarios. For example the MoveSmarter platform can be used together with the Wish List API. It's up to you!

MoveSmarter consists of a mobile sensing lib and a server platform. The sensing lib can be embedded in one's own mobile application (Apple or Android), and it autonomously senses all trips the user makes and submit them to the MoveSmarter back-end platform which further analyses and enriches these trips, creates long-term personal mobility patterns of all sorts and kinds and makes them available for Apps or mobile applications in a secure way. This results in a rich personal dataset which can be queried using a REST/JSON style API.

Information items that can be accessed via the interface are:

  • timestamps of each trip start and end;

  • location mapping of trip origin, route and destination;

  • modality of each trip;

  • frequent visited personal places;

  • frequent routes;

  • Moreover via the interface combinations of information items can be retrieved. For example

  • give me all trips of user X this week

  • give me all users that have visited location Y in the last 24 hours

  • give me all users that have visited location Z more than three times since the start of using the App AND stayed their for more than 3 hours each time.

To get access to this API please follow the procedure below:

  • The data model as well as the API specification are documented at https://www.movesmarter.nl/portal/appdev/test. Java stubs for all classes are also provided in Mobidot_java_classes.zip which can be used for easy serialization. The java classes zip file can be requested from the Mobidot contact person together with the sensing lib.

Please follow these steps to start gathering mobility data and to access this enriched data via the MoveSmarter API:

  • Register a new app under https://www.movesmarter.nl/portal/appdev/and use the sensing lib by sending a mail to the contact person of Mobidot. Alternatively, you could also request a version of the existing MoveSmarter Beta App which already has the sensing lib embedded.

  • Log in with your username and credentials, and mention 3cixty as domain and start collecting data.

  • Read the documentation at https://www.movesmarter.nl/portal/appdev/test where you can both testdrive specific API calls and make yourself familiar with the API call signatures, inputs and outputs. There is both an oAuth-based and an API key-based access method, both are described under the developer tab. These APIs can be extended if you have specific requests for data that is available but not yet exposed via these interfaces.

    • IdentityManager: manages users, groups and domains

    • MeasurementService: stores the actual trip and location data

    • PersonalMobility: analyzes trip data and creates long-term personal and community patterns of mobility

    • IncentiveManager: allows to provide timely and targeted incentives based on mobility behaviour (not used for 3cixty)

  • New users and groups can be added at https://www.movesmarter.nl/portal/domainadmin/

  • Personal mobility information can be inspected at the Mobility Footprint tab, like the list of trips at https://www.movesmarter.nl/portal/personal/trips

Documentation is available at https://www.movesmarter.nl/portal/appdev/test where you can both testdrive specific API calls and make yourself familiar with the API call signatures, inputs and outputs. There is both an oAuth-based and an API key-based access method, both are described under the developer tab. Note that you will need to create a separate developer login there.

Normal way of working is to:

  • Use the provided java classes (in the Mobidot_java_classes.zip file)

  • Do a call on the Movesmarter API

  • Serialize the JSON response to the appropriate java classes

  • Use the information from the obtained java instance

See the "MobidotIntegrationAPI" document in the 3cixty API docs for some shortcuts on using the above API with your 3cixty related apps.

The sensing lib is available on request from the Mobidot [contact person](johan.koolwaaij@mobidot.nl). Note that you are not allowed to redistribute it to other persons or parties. The sensing lib is available for both Android and iPhone. The documentation of the sensing lib is also provided in the documentation file attached to the sensing lib (named Mobidot_sensing_lib_doc.rar).

The MoveSmarter Measurement API runs at:

  • https://www.movesmarter.nl/external/measurement (external)

  • https://www.movesmarter.nl/api/measurement (oAuth)

The MoveSmarter PersonalMobility API runs at:

  • https://www.movesmarter.nl/external/personalmobility (external)

  • https://www.movesmarter.nl/api/personalmobility (oAuth)

The MoveSmarter sensing lib comes as a native library including documentation of its API and a usage example.
